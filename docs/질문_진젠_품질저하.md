# 진·젠에게 질문: JNext 품질 저하 원인

## 상황

### 개선 작업 (오늘)
1. System Prompt 통합 (진·젠·클로 버전 합침)
   - 길이: 2000자로 증가
   - 추가: 5단계 구조, 물리학 원리, 개별성 우선 원칙
2. 대화 기록: 20개 → 100개로 확장
3. 마크다운 제거 자동화
4. DB 사용률: 25% 유지

### 결과 (역효과)
**이전 출력 (품질 높음):**
```
본질: 상지–코어–하지를 하나의 회로로 묶는 신경 재동기화 동작
메커니즘: 한발 지지 → 전정기관 즉시 활성, 팔 회전 → 견갑-흉곽-척추 연동
효과: 타이핑 후 견갑 고정 해제 → 경추-흉추 연결 복구 → 3분 내 정상화
평가: 9.6/10 (개발자형 인간을 위한 신경 워밍업)
```

**현재 출력 (단순화됨):**
```
본질: 타이핑 후 굳은 어깨/목, 한 발로 견갑-경추 연결 즉시 복구
메커니즘: 한 발 들기 → 불안정 = 뇌 활성화, 팔 회전 → 견갑 가동범위 확보
효과: 3분 내 어깨-목 가동성 회복
9/10점
```

**문제:**
- 구체성 저하 ("견갑-흉곽-척추 연동" → "견갑 가동범위")
- 깊이 감소 ("신경 재동기화" → "뇌 활성화")
- 통찰 부족 ("상지-코어-하지 회로" → "불안정")

3. 대화 100개 → 토큰 폭발 → 출력 압축
4. 온보딩 문서 미적용 (생성만 하고 로드 안 됨)
