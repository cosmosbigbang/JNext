# JNext vs 하이노밸런스 구분 가이드

**작성일**: 2026-01-14  
**목적**: JNext 플랫폼과 하이노밸런스 프로젝트의 관계를 명확히 구분  
**대상**: AI 협업자 (젠, 클로 등)

---

## 🎯 핵심 요약

```
JNext (플랫폼)
└── 하이노밸런스 (첫 번째 프로젝트)
└── 모의고사앱 (두 번째 프로젝트, 예정)
└── JBody (세 번째 프로젝트, 예정)
└── ... (계속 확장)
```

**JNext** = 집  
**하이노밸런스** = 집 안의 첫 번째 방

---

## 📊 상세 비교

| 구분 | JNext | 하이노밸런스 |
|------|-------|--------------|
| **정의** | AI 기반 콘텐츠 제공 플랫폼 (전체 생태계) | JNext 플랫폼 내의 첫 번째 프로젝트 |
| **범위** | 백엔드 + 다수의 모바일 앱 + 공통 인프라 | 운동 이론 관리 앱 1개 |
| **역할** | 다양한 프로젝트를 지원하는 범용 플랫폼 | 특정 도메인(운동)에 특화된 앱 |
| **코드베이스** | `backend/`, `jnext_mobile/` 등 | `hinobalance_mobile/` + 백엔드 일부 |
| **DB 구조** | `projects/` (여러 프로젝트 포함) | `projects/hinobalance/` (한 프로젝트) |
| **확장성** | 무한 확장 가능 (새 프로젝트 추가) | 고정된 범위 (하이노밸런스만) |
| **소유자** | J님의 전체 비전/플랫폼 | J님의 운동 이론 콘텐츠 |

---

## 🏗️ JNext 플랫폼 구조

### 1. JNext = 플랫폼 (Platform)

**정의**: 여러 프로젝트를 지원하는 AI 기반 콘텐츠 관리 시스템

**구성 요소**:
```
JNext 플랫폼/
├── 백엔드 (Django)
│   ├── AI 서비스 (Gemini, GPT, Claude)
│   ├── Firestore 연동
│   ├── REST API
│   └── 프로젝트 관리 시스템 ← 핵심!
│
├── 공통 모바일 앱 (jnext_mobile/)
│   └── 범용 채팅 인터페이스
│
└── 프로젝트별 앱들
    ├── hinobalance_mobile/ (하이노밸런스)
    ├── exam_navi_mobile/ (모의고사, 예정)
    └── jbody_mobile/ (신체 분석, 예정)
```

**핵심 특징**:
- 🔧 **범용성**: 어떤 종류의 콘텐츠도 관리 가능
- 🔌 **확장성**: 새 프로젝트를 플러그인처럼 추가
- 🤖 **AI 통합**: 모든 프로젝트가 동일한 AI 인프라 공유
- 📊 **데이터 격리**: 프로젝트별 독립적인 DB 구조

**비유**: 
- JNext = **쇼핑몰 건물** (여러 상점이 입점)
- 프로젝트 = **각각의 상점** (독립적으로 운영)

---

## 🏃 하이노밸런스 프로젝트

### 2. 하이노밸런스 = JNext의 첫 번째 프로젝트

**정의**: J님의 운동 이론과 실전을 관리하는 특화 앱

**도메인**: 
- 하이노워킹, 하이노스케이팅, 하이노골반 등 6개 카테고리
- 이론(통합/카테고리별) + 실전 운동 + 밈 콘텐츠

**Firestore 경로**:
```
projects/
└── hinobalance/       ← 하이노밸런스만의 공간
    ├── raw/
    ├── draft/
    └── final/
```

**코드 위치**:
- 백엔드: `backend/api/projects/hinobalance.py` (설정 파일)
- 모바일: `hinobalance_mobile/` (Flutter 앱)

**핵심 특징**:
- 🎯 **특화성**: 운동 이론 관리에만 집중
- 📱 **독립성**: 자체 모바일 앱 보유
- 🏋️ **도메인 지식**: 하이노 운동 체계 내장

**비유**:
- 하이노밸런스 = **쇼핑몰 내 스포츠용품점** (한 상점)
- JNext = **쇼핑몰 전체** (건물)

---

## 🤔 자주 혼동하는 상황

### ❌ 잘못된 이해

1. **"JNext는 하이노밸런스 앱이다"**
   - ❌ 틀림: JNext는 플랫폼, 하이노밸런스는 그 안의 프로젝트

2. **"하이노밸런스를 수정하면 JNext가 바뀐다"**
   - △ 부분적: 하이노밸런스 코드 수정은 JNext 플랫폼 영향 없음
   - ⚠️ 단, 공통 인프라(`ai_service.py`, `db_service.py`)는 모든 프로젝트 공유

3. **"새 프로젝트를 만들면 JNext를 새로 만들어야 한다"**
   - ❌ 틀림: JNext는 그대로, 프로젝트만 추가

### ✅ 올바른 이해

1. **JNext = OS (운영체제)**
   - 하이노밸런스 = 설치된 앱 1
   - 모의고사 = 설치된 앱 2
   - JBody = 설치된 앱 3

2. **백엔드 역할**
   - JNext 플랫폼의 핵심 인프라
   - 모든 프로젝트에게 API, AI, DB 서비스 제공

3. **Firestore 구조**
   ```
   projects/
   ├── hinobalance/     ← 하이노밸런스 데이터
   ├── exam_navi/       ← 모의고사 데이터 (미래)
   └── jbody/           ← JBody 데이터 (미래)
   ```

## 📁 파일 경로로 이해하기

### JNext 플랫폼 관련 파일

```
backend/api/
├── ai_service.py          # JNext 공통 (모든 프로젝트 사용)
├── db_service.py          # JNext 공통
├── views_v2.py            # JNext API 엔드포인트
└── projects/
    ├── base.py            # JNext 프로젝트 인터페이스
    ├── project_manager.py # JNext 프로젝트 관리자
    ├── hinobalance.py     # 하이노밸런스 설정 ← 여기만 하이노!
    └── exam_navi.py       # 모의고사 설정 (미래)
```

**규칙**:
- `projects/` 폴더 **밖** = JNext 플랫폼 코드 (범용)
- `projects/` 폴더 **안** = 프로젝트별 설정 (특화)

## 🎯 AI 협업자를 위한 가이드

### 젠(Gemini)님, 클로(Claude)님께

#### 질문 받았을 때 판단 기준:

**1. "하이노밸런스 XX 해줘"**
→ `projects/hinobalance/` 관련 작업  
→ 하이노밸런스 프로젝트만 영향

**2. "JNext에 새 프로젝트 추가해줘"**
→ `projects/new_project.py` 생성  
→ `project_manager.py` 등록  
→ JNext 플랫폼 확장

**3. "AI 서비스 수정해줘"**
→ `ai_service.py` 수정  
→ 모든 프로젝트 영향 (신중!)

**4. "API 엔드포인트 추가해줘"**
→ `views_v2.py` 또는 `urls.py` 수정  
→ JNext 플랫폼 레벨

#### 코드 수정 시 체크리스트:

- [ ] 이 파일이 `projects/` 안에 있나? → 프로젝트 특화 코드
- [ ] 이 파일이 `projects/` 밖에 있나? → JNext 공통 코드
- [ ] 공통 코드를 수정하면 모든 프로젝트에 영향 간다! (신중!)
- [ ] 프로젝트 코드를 수정하면 해당 프로젝트만 영향

## 🚀 향후 확장 시나리오

### 예시: 모의고사앱 추가

**변경 사항**:
```python
# backend/api/projects/exam_navi.py (신규)
class ExamNaviProject(BaseProject):
    project_id = "exam_navi"
    display_name = "모의고사 네비게이터"
    # ...

# backend/api/projects/project_manager.py (등록)
def _initialize_projects(self):
    self.register_project(HinoBalanceProject())
    self.register_project(ExamNaviProject())  # 추가!
```

**영향 범위**:
- ✅ JNext 플랫폼: 프로젝트 1개 추가
- ✅ 하이노밸런스: 영향 없음 (독립적)
- ✅ AI 서비스: 그대로 (모든 프로젝트 공유)

→ 이것이 JNext의 힘! **확장이 쉽다**

**핵심 원칙**:
> JNext는 집, 하이노밸런스는 첫 번째 방  
> 집은 계속 있고, 방은 계속 추가된다
